Frameworks

Lightweight

All WickedFlame components leave no trace in client code. Components can be capsulated to destinct places where they can easily be replaced.

Fluent

All WickedFlame components use a Fluent Synthax that is simple to use. The Fluent API allways leads the user to the desired result.

Configuration

WickedFlame components can be configured in the code or within the application config file

Simple API

WickedFlame components are designed to have a simple and intuitive API that is easy to use and understand.

PersistanceMap

Persistance

PersistanceMap is a small, extremely lightweight and intuitive code fist, convention based ORM Framework for .NET.

Queries

Queries are defined using a Fluent API that compiles to SQL. The compiled SQL is executed against a RDBMS using the underlying ADO.NET.
The returned data is automaticaly projected into the desired Dataobjects/POCO's.

Simple

The synthax is simple and easy to use and resemles the SQL or LINQ synthax.

Lightweight

PersistanceMap is extremely lightweight and leaves no traces in the client code.

Supported Databases

PersistanceMap currently supports MSSql and SQLite RDMBS Servers.

Installation

PersistanceMap can be installed from NuGet through the package manager console.



Installation

Inside VisualStudio launch the NuGet Package Manager Console
MSSql Server 
PM > Install-Package PersistanceMap
SQLite 
PM > Install-Package PersistanceMap.Sqlite

Examples

MSSql Database 
var provider = new SqlContextProvider(connectionString);
using (var context = provider.Open())
{
    // use context to create Queries
    context.From<Order>()
	.Join<OrderDetail>(od, o) => od.OrderID == o.ID)
	.Select<OrderWithDetail>()
}
Sqlite Database 
var provider = new SqliteContextProvider(connectionString);
using (var context = provider.Open())
{
    // use context to create Queries
    context.From<Order>()
	.Join<OrderDetail>((od, o) => od.OrderID == o.ID)
	.Select<OrderWithDetail>()
}
